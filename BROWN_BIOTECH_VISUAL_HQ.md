# Brown Biotech Visual HQ

## Purpose
Show the business at a glance so the team can answer:
- What came in?
- Who owns it?
- What happens next?
- Does it need approval?
- What should be reused?

## Visual layers
### 1. Intake flow
A left-to-right flow from request to receipt.

**Stages:**
Signal → Qualify → Brief → Route → Review → Deliver

### 2. Service lanes
A simple lane board for:
- peptide-service
- biostatx
- genox-site
- business-pipeline

### 3. Proof board
A compact row of:
- research proof
- publications
- case notes
- partner updates

### 4. Weekly control panel
A small status block with:
- wins
- blockers
- risks
- decisions
- next week priorities

### 5. Approval gate
A visible block for:
- money
- contracts
- deployment
- public claims
- medical claims

## Recommended layout
- **Top:** one-line strategy
- **Left:** intake + lanes
- **Center:** current requests and owners
- **Right:** proof + approvals
- **Bottom:** weekly review + next actions

## What it should feel like
- clean
- premium
- decision-ready
- low-noise
- easy to read in under a minute

## What to show on the screen
- primary motion
- current active lane
- active owner
- next action
- approval status
- proof asset linked to the request

## What not to show
- long prose
- duplicate documents
- deep internal notes
- every possible lane at equal weight
- anything that weakens the primary motion

## Build rule
If a visual element does not help intake, routing, delivery, proof, or review, leave it out.
