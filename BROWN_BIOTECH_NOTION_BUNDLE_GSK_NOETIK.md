## 2) 10-person operating matrix
**Purpose:** keep the team operating like a small business system, not a discussion group.

### Operating rule
Every role owns:
- one input
- one output
- one handoff
- one metric

### Matrix

| Role | Primary input | Core output | Weekly KPI |
|---|---|---|---|
| Strategy / approval | market signals, priorities | approved direction | decisions made |
| Intake | inbound requests | structured intake records | intake completion rate |
| Triage | intake records | urgency + fit assessment | triage turnaround time |
| Routing | triage outcomes | assigned owner + lane | routing accuracy |
| Peptide-service | paid briefs | decision-ready delivery | briefs delivered |
| Biostatx | analysis requests | analysis/report package | analysis turnaround |
| Genox-site | discovery/partner signals | partner-facing framing | qualified opportunities |
| Business-pipeline | ops requests | automation / routing support | flow improvements shipped |
| Proof / publications | completed work | case note / proof asset | proof items published |
| QA / launch checks | drafts, releases | verified release readiness | issues caught before launch |

### Cadence
#### Daily
- review new intake
- assign owner
- confirm next action

#### Weekly
- review wins
- review blockers
- review risks
- review proof
- decide what to reuse

#### Monthly
- check lane balance
- refine pricing and routing
- prune low-value work
- update the operating matrix

### Team success criteria
The team is healthy if it can reliably do the following:
1. convert inbound interest into a brief
2. convert briefs into routed delivery
3. convert delivery into proof that improves the next close

### Management note
If a task does not improve intake, routing, delivery, proof, or review, do not add it.
