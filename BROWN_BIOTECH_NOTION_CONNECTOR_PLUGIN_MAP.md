# Brown Biotech Notion Connector / Plugin Map

## Goal
Match the Claude legal-stack pattern with Brown Biotech equivalents.

## Connectors
These systems feed live work into Brown Biotech HQ.

### Core connectors
- Notion HQ
- GitHub
- Telegram
- Vercel
- Google Drive / PDFs
- Browser-based site tests

### Research connectors
- PubMed / PMC
- GEO / SRA
- TCGA
- Hugging Face Datasets
- bioRxiv / medRxiv sources

### Evidence connectors
- dataset watcher outputs
- QC notebooks / artifacts
- research briefs
- publication drafts

## Plugin / lane map
These are the Brown Biotech equivalents of practice-area plugins.

### 1) Paid Brief Intake
- classify request
- check fit
- route to owner
- prepare receipt

### 2) Service Routing
- map request to lane
- apply escalation rules
- flag approval needs
- produce next action

### 3) Dataset Watcher
- scan sources
- score hits
- separate ingest / QC / archive
- write reviewed outputs into Research Proof

### 4) QC Review
- validate schema and metadata
- summarize quality issues
- decide ingest vs revise vs archive

### 5) Decision Memo
- create a decision-ready summary
- list evidence
- state recommendation
- note risks and approvals

### 6) Weekly Review
- summarize wins, blockers, risks
- surface decisions needed
- identify reusable templates

### 7) Launch Check
- claims review
- deployment readiness
- rollback note
- approval gate

### 8) Partner Brief
- capture partner fit
- summarize value exchange
- assign owner
- create follow-up action

### 9) Publication Pack
- draft blog or thread
- include proof and next action
- hold for review when needed

## Setup interview fields
Every connector or plugin should know:
- owner
- source of truth
- output format
- escalation path
- approval gate
- refresh cadence

## Brown Biotech priority order
1. Paid Brief Intake
2. Service Routing
3. Weekly Review
4. Dataset Watcher
5. QC Review
6. Decision Memo
7. Launch Check
8. Partner Brief
9. Publication Pack

## Recommendation
Start with the first three because they create the company operating loop.
