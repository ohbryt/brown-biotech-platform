# Brown Biotech Notion Migration Checklist

## Goal
Move the current operating system into Notion without losing routing, ownership, or approval clarity.

## Step 1 — Create root pages
- Brown Biotech HQ
- Service Intake Templates
- Intake Requests
- Intake Routing
- Funnel
- Weekly Reviews
- Decision Log
- Launch Checks
- Skill Pack
- Research Proof
- Publications
- Community Updates
- Partner Funnel
- 30-Day Plan
- Team Roster
- Team Strategy
- Team Operating Pack
- Week 1 Execution Script
- Role Deliverables
- DB Field Spec
- Operating Slide

## Step 2 — Add database properties
- Use the field spec as the source of truth
- Keep title fields short and visible
- Add only properties that support routing, approval, or follow-up

## Step 3 — Set page relations
- Intake Requests ↔ Intake Routing
- Intake Requests ↔ Funnel
- Weekly Reviews ↔ Decision Log
- Publications ↔ Research Proof
- Team Strategy ↔ Team Operating Pack

## Step 4 — Add templates
- One intake template per lane
- One weekly review template
- One decision log template
- One publication draft template
- One partner funnel template

## Step 5 — Validate the flow
For one sample request, confirm:
- request ID exists
- lane is assigned
- owner is assigned
- approval flag is set
- next action is visible
- receipt is reusable

## Step 6 — Freeze the language
- Request a Brief
- one brief
- one owner
- one next action
- human approval for high-stakes actions

## Done when
- the HQ page is readable in under a minute
- the intake path is visible
- the team knows where to check weekly work
- the same language appears in the website and Notion
