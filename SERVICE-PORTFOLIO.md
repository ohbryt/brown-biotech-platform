# Brown Biotech Service / Site Portfolio

This document groups the Brown Biotech ecosystem sites into clear lanes so the team can manage them without mixing strategy, branding, and experiments.

## Portfolio rules
- `brownbio.tech` is the canonical main site.
- `services.brownbio.tech` is the canonical service hub.
- Each additional site should have one primary job.
- If a site cannot be described in one sentence, it is probably doing too much.
- Legacy or experimental sites should be marked clearly and either upgraded, merged, or retired.

| Site / project | Lane | Purpose | Recommended status |
|---|---|---|---|

## How to manage them

### 1) Keep main vs service separate
- Main site = company identity, authority, and top-of-funnel trust.
- Service hub = offers, productized services, lead capture, and conversion flows.

### 2) Put experiments behind a decision rule
Use one of these outcomes for every experimental site:
- **Promote**: it has traction and becomes a real service.
- **Merge**: the idea fits an existing site better.
- **Retire**: it is no longer needed.

### 3) Standardize each site's one-liner
Every site should have:
- one sentence purpose
- primary audience
- primary CTA
- owner
- status

### 4) Review cadence
- Weekly: what changed, what launched, what converted
- Monthly: merge/retire decisions
- Quarterly: re-map domains and keep only the useful surface area
